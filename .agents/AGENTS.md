# Agent Rules: 『強いSEO』に基づくSEO品質基準

当プロジェクト（markdown-viewer）の開発・変更において、書籍『強いSEO』のナレッジを開発プロセスに永続的に組み込み、常に検索エンジンとユーザー体験に強いコードを出力するためのガイドラインです。

---

## 🔍 1. テクニカルSEO・クローラビリティ基準
コード変更、タグ追加、構造変更を行う際は、以下のクローラビリティ要件を維持・強化してください。

- **URL正規化 (Discover / クロール対策)**:
  - 重複コンテンツ判定を防ぐため、HTMLファイルには常に `<link rel="canonical" href="https://markdown-viewer.net/">` を配置または維持する。
  - 新しいページやコンテンツが追加された場合は、`sitemap.xml` および `robots.txt` の整合性を必ず更新する。
- **セマンティックなHTML構造**:
  - 検索エンジンに文書構造を正確に解析させるため、見出しタグ（`h1`〜`h3`）の適切な親子階層、箇条書き（`ul`/`ol`/`li`）、段落（`p`）、太字（`strong`）を正しく用いてマークアップする。
  - 見出しタグのジャンプ（例: `h1` の直後に `h3` が来るなど）を避け、セマンティックなマークアップルールを遵守する。
- **構造化データの維持**:
  - ページ内にある JSON-LD（`WebApplication` スキーマなど）の記述を破壊せず、サービス内容の変更に合わせて常に正確なメタ情報を維持・拡張する。

---

## ⚡ 2. ユーザビリティ・Core Web Vitals基準
ページの表示パフォーマンスおよび表示安定性は、Googleの評価アルゴリズム（特にランキングプロセス）に直結するため、コード品質として担保します。

- **CLS (Cumulative Layout Shift) の徹底防止**:
  - 広告コンテナ（`.adsense-container`）、画像、埋め込み要素など、読み込みに遅延が発生する動的要素に対しては、CSSで必ず最小の高さ（`min-height`）や固定アスペクト比を事前に確保し、ロード後のガタつきをゼロにする。
- **コンソールエラーの徹底排除**:
  - ページの描画遅延やクローラーによるレンダリング失敗を防ぐため、表示タイミング（モバイル版のタブ切り替え時など）に合わせたスクリプトの実行制御（Lazy Loadなど）を行い、実行時エラー（例: `availableWidth=0`）を絶対に発生させない。
- **モバイルフレンドリーの厳守**:
  - モバイル端末でのタップターゲットサイズ（`min-width`/`min-height` は44px以上）、適切なビューポート設定、文字の読みやすさ（十分なコントラスト、行高 `1.6`〜`1.8`）を維持する。

---

## 📝 3. コンテンツの独自性（E-E-A-T）基準
ドキュメントやテンプレート、初期テキスト（`sampleMarkdown` など）の生成・追加を行う際の品質基準です。

- **独自性（インプット・スループット・アウトプットモデル）の適用**:
  - ネット上のありふれたコピペ記事にならないよう、実体験（Experience）や開発者の一次情報（例: IDEでの利用、プロンプトテンプレート、LLMトークン節約手法）をベースとする（**インプットのずらし**）。
  - 事実だけでなく、なぜそれが重要なのかという技術的な解釈や示唆を追加する（**スループットの磨き込み**）。
  - リスト、テーブル、コードブロックなどの要素を適切に使い、ユーザーにとって最も可読性の高い形式にする（**アウトプットの最適化**）。
- **ターゲットキーワードの自然な網羅**:
  - `マークダウン` などの基本キーワードに加え、開発者が検索する `AI活用`, `生成AI`, `Cursor`, `VS Code`, `プロンプト` などの周辺コンテキストキーワードを、文書構造の中に自然に盛り込む。
